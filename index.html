FURNACE

We studied what users love about every agent harness and built it all into Furnace.
Built in the open, shaped by the people who use it.

×

What Furnace does

  • Terminal UI Interactive Ink-based TUI with streaming output, autocomplete, and a keyboard-driven settings panel.
  • Sessions & Forks Local SQLite sessions stored as append-only entry trees. Resume, fork, or clone without losing context.
  • Typed Tools Read, search, edit, write, and run bounded shell commands through a typed tool registry.
  • Permission Gates Allow, ask, or deny each risky tool call. Session-scoped grants and plan-mode clamps keep you in control.
  • Context Compaction Model-assisted summaries and Headroom-lite compression keep long sessions within context limits.
  • Skills Progressive-disclosure instruction packages. Reuse Cursor, Claude Code, and custom skill roots.
  • Subagents Delegate independent work through a normal task tool, with foreground and background task groups.
  • Plan Mode Read-only planning workflow that writes durable plan artifacts before switching to implementation.
×

First run

npm install -g cook-furnace furnace

Interactive mode starts by default. Use /login to configure your provider, then type prompts and review tool approvals.

# Or pass a prompt directly furnace -p "Summarize this repository"

Requires Node 22+. Supports any OpenRouter-compatible model. Session state is stored locally — no cloud required.
